1. תקציר מנהלים
--------------------------------------------------------------------------------
MP Smart Analyzer הוא כלי MVP שרץ במלואו בדפדפן של המשתמש (לקוח בלבד).
המשתמש מדביק "דוח מצב" בטקסט חופשי; מנוע חוקים דטרמיניסטי מנתח את הטקסט,
בוחר תבנית פלט, ומייצר מסמך Markdown מוכן להעתקה או להורדה.

אין שרת יישום של המוצר, אין קריאות API חיצוניות לניתוח, ואין העלאת דוחות
לענן כחלק מהלוגיקה — הניתוח מתבצע מקומית בדפדפן.

מאפיין מרכזי: שקיפות והסבר ניתן (matched rules, signals, scores) לעומת
מודל שפה חיצוני שלא נכלל בפרויקט זה.


2. מטרת המוצר וקהל יעד
--------------------------------------------------------------------------------
- מהירות: קלט → ניתוח → פלט מובנה (Markdown).
- שימוש טיפוסי: הכנת תקצירים, הוראות, דוח ללקוח, צ'ק-ליסט — לפי תבנית שנבחרה.
- קהל: מפתחים, מובילי צוות, אנשי מוצר שמעבירים "דוח מצב" טקסטואלי לפורמט עבודה.

4. רכיבי מערכת (לוגיקה)
--------------------------------------------------------------------------------

4.1 rules.js — מנוע החוקים
- RULES: מערך חוקים עם whenAny / whenAll, משקלי תבניות, ואותות (signals).
- normalize(): נרמול טקסט (אותיות קטנות, RLM/LRM, ריווחים).
- evaluateRules(): הרצת חוקים, צבירת scores, matchedRules, facts
  (טכנולוגיות, שורות בעיה, action hints, סוג קלט).
- pickTemplate(): בחירת תבנית לפי ציונים + confidence.
- decide(): חומרה, מצב תגובה, טון אוטומטי, תעדוף (קריטי/חשוב/מאוחר יותר),
  המלצה לסוג פלט.
- assessInputQuality / assessReportUsability: בקרת איכות קלט לפני פלט "מקצועי".

4.2 templates.js — תבניות פלט
- תבניות בעברית ובאנגלית (אובייקטים HE / EN).
- render() לכל תבנית עם tone, facts, decision וכו'.
- getInvalidInputMarkdown(): פלט כאשר הקלט נדחה בשלב איכות.
- verifyTemplateLocaleParity(): בדיקת התאמה בין מפתחות תבניות בשפות (במסגרת npm test).

4.3 intelLayer.js — שכבת "הבנה" (פרשנות, לא LLM)
- טקסט מסכם לפני הפלט: סוג קלט, מוקדי בעיה, ביטחון, כיוון מומלץ,
  מסלול החלטה תמציתי.
- מותאם למצב נוער (מפורט יותר) מול בוגרים (תמציתי).
- מסווג "טעם דוח": טכני / עסקי / כללי — לפי אותות ומילות מפתח.

4.4 app.js — אורקסטרציה ו-UI
- טעינת מודולים (ESM), אלמנטי DOM, מחרוזות דו-לשוניות.
- computeResponse(): זרימה מלאה — איכות קלט → תבנית → Markdown (כולל summary מסונן).
- analyze(): ניתוח בלחיצה בלבד (ללא ניתוח אוטומטי בכל שינוי select).
- שכבת הבנה: setIntelBrief(buildIntelBriefMarkdown(...)).
- העתקה/הורדה: כולל שכבת הבנה + מפריד --- + פלט ראשי.
- מצבים: נוער/בוגרים, מהיר/עשיר, מבט פנימי/חיצוני/נטרלי (נשמר ב-localStorage).
- debug: פרמטר ?debug ב-URL — JSON מפורט ב-details.


5. ממשק משתמש (index.html + style.css)
--------------------------------------------------------------------------------
- עיצוב כהה ("קונסולת ניתוח"): כחול חשמלי, ירוק מערכת, סגול עדין.
- כותרת: שם מוצר, תג סלוגן, סטטוס "פעיל", מצב נוער/בוגרים, שפת ממשק,
  דוגמה/ניקוי.
- פאנל קלט: Data Input, שלבי זרימה (במצב נוער), מדריך, textarea עם רשת עדינה,
  מונה תווים ותג אורך.
- פאנל פלט: Analysis Output, מטריקות (מצב מזוהה, ביטחון), שורת תבנית,
  שכבת הבנה (intel), פלט ראשי, Copy/Download, debug מתקפל.
- אזור הגדרות: "הגדרות מערכת" + מבט (פנימי/חיצוני/נטרלי) + שפת פלט, סוג פלט, טון.
- כפתור מרכזי: הרץ ניתוח / Run Analysis.
- מובייל: טאבים קלט/פלט.
- quick-mode: פחות אנימציה וצל; rich-mode: פלאש הצלחה עדין וכו'.


6. הרצה, בדיקות, תלויות
--------------------------------------------------------------------------------
הרצה מומלצת: שרת סטטי מקומי (ES modules; file:// עלול לחסום import בחלק מהדפדפנים).

  npm run dev:view    →  npx serve על פורט 3000 (לפי package.json)
  npm test            →  node --test tests/rules.test.js (דורש Node ב-PATH)

אין תלות runtime ב-package.json מלבד סקריפטים; אין node_modules כחלק מהמוצר.


7. אבטחה, פרטיות, שקיפות
--------------------------------------------------------------------------------
יתרון: תוכן הדוח לא נשלח לשרת היישום של הכלי — מתאים לנתונים רגישים.
אזהרה: במצב debug, פירוט ב-console/במסך עלול לחשוף תוכן קלט למי שרואה את המסך.

הכלי אינו מציג כ"הוא מודל AI חיצוני"; שכבת intel מבוססת כללים ומחרוזות קבועות.
