---

## 1. תקציר מנהלים

**MP Smart Analyzer** הוא כלי MVP מקומי (ללא שרת) שמקבל “דוח מצב” בטקסט חופשי, מריץ מנוע חוקים דטרמיניסטי, בוחר תבנית פלט, ומייצר Markdown מוכן להעתקה או הורדה. הממשק דו־לשוני (עברית/אנגלית), RTL/LTR, ומותאם למובייל (טאבים קלט/פלט).

**מסקנה:** המוצר מתאים היטב ליעד MVP — פשוט, שקוף, וניתן להרחבה דרך `rules.js` ו־`templates.js`. אין בינה מלאכותית חיצונית; כל הניתוח מתבצע בדפדפן.

---

## 2. מבנה הפרויקט ותלויות

| קובץ | תפקיד |
|------|--------|
| `index.html` | מבנה DOM, נגישות בסיסית (`aria-*`), טעינת `app.js` כ־module |
| `app.js` | אורקסטרציה: UI, אנימציית פלט, העתקה/הורדה, debug |
| `rules.js` | נרמול טקסט, חוקים, ציונים לתבניות, `decide()`, `pickTemplate()` |
| `templates.js` | תבניות HE/EN, רינדור Markdown |
| `style.css` | עיצוב, רספונסיביות, `prefers-reduced-motion` בלוגיקת אנימציה (דרך JS) |

**תלויות חיצוניות:** אין `package.json`, אין build chain — רק ES modules מקומיים.  
**הרצה:** פתיחת `index.html` בדפדפן (מומלץ דרך שרת סטטי אם יש בעיות CORS עם modules בחלק מהדפדפנים בפתיחה מ־`file://`).

---

## 3. זרימת נתונים (Pipeline)

1. **קלט:** טקסט ב־`#report` + בחירות: שפת UI, שפת פלט, טון, סוג פלט.
2. **`evaluateRules(reportText)`** (`rules.js`):
   - נרמול (אותיות קטנות, RLM/LRM, מרכאות).
   - התאמת חוקים (`whenAny` / `whenAll`) והוספת משקלים ל־`scores`.
   - חילוץ **facts**: טכנולוגיות, שורות “בעיה”, רמזי פעולה, סוג קלט (`inputType`).
3. **`pickTemplate(scores)`:** בחירת תבנית לפי ציון מקסימלי + **confidence** נגזר.
4. **`decide(analysis, templateKey)`:** חומרה, מצב תגובה, טון אוטומטי, תעדוף (קריטי/חשוב/בהמשך), והמלצה לסוג פלט.
5. **`computeResponse()`** (`app.js`): התאמת `outputLang`, `outputType`, בחירת מפתח תבנית (כולל override ל־`client_report` / `checklist`).
6. **`template.render(...)`** → Markdown; במצב `summary` — סינון כותרות דרך `toSummaryMarkdown()`.
7. **פלט:** `typeOutput()` (הקלדה מדומה או מיידי לפי אורך / reduced motion).

**Debug:** `?debug` ב־URL מוסיף לוגים ל־console; פירוט JSON ב־`<details>`.

---

## 4. נקודות חוזק

- **דטרמיניזם ושקיפות:** אותו קלט → אותו פלט; ניתן להסביר החלטות דרך `matchedRules`, `scores`, `signals`.
- **הפרדת אחריות:** חוקים נפרדים מתבניות — קל להוסיף חוק או תבנית בלי לשבור את ה־UI.
- **תמיכה דו־לשונית:** מחרוזות UI, כיוון מסמך, ותבניות EN חלקיות (HE עשיר יותר).
- **UX:** מונה תווים, תג אורך, טוסט העתקה, מצבי שגיאה/הצלחה בפאנלים, טאבים במובייל.
- **נגישות:** `aria-live` בסטטוס, `focus-visible` ב־CSS, תוויות ל־textarea.
- **ביצועים:** אנימציית פלט מושבתת לטקסט ארוך (>6000 תווים) או כש־`prefers-reduced-motion: reduce`.

## 5. מגבלות וסיכונים

### 5.1 מוצר ולוגיקה

- **אין הבנה סמנטית עמוקה** — רק מילות מפתח ודפוסי שורות; דוחות עמומים ייפלו ל־`general` עם confidence נמוך יותר.
- **`pickTemplate`:** confidence מחושב כיחס ציון מוביל לסכום + offset; כשהתחרות צמודה, הבחירה עלולה להרגיש “אקראית” למשתמש למרות שהיא דטרמיניסטית.
- **תבניות באנגלית:** ב־`templates.js`, אובייקט `EN` חסר מפתחות מלאים (למשל אין `spa_upgrade`, `fullstack_polish`, `client_report` כמו ב־HE). בחירת שפת פלט English עלולה להחזיר fallback ל־`general` או לזרוק שגיאה אם המפתח חסר — כדאי ליישר קו בין השפות.

### 5.2 טכני ותפעול

- **ES modules מ־`file://`:** בחלק מהדפדפנים/מדיניות אבטחה, ייבוא מקומי עלול להיכשל; מומלץ להריץ עם `npx serve` או שרת סטטי קל.
- **אין בדיקות אוטומטיות:** אין unit tests ל־`rules.js` / `decide` — רגרסיות בחוקים קשות לאיתור.
- **אין גרסה/build:** אין מספור גרסאות ב־UI או בקבצים.

### 5.3 אבטחה ופרטיות (יתרון + הערה)

- **יתרון:** אין שליחת דוחות לשרת — מתאים לתוכן רגיש.
- **הערה:** לוגים ב־console במצב debug עלולים לחשוף תוכן דוח אם מישהו מסתכל על המכשיר.

## 6. נגישות ו־UX — פירוט

| נושא | מצב |
|------|-----|
| כיוון ושפה | `lang` ו־`dir` מתעדכנים לפי בחירת שפת UI |
| סטטוס דינמי | `aria-live="polite"` על `#status` ו־`#toast` |
| מקלדת | `:focus-visible` על כפתורים, טאבים, select, textarea |
| מובייל | פאנלים מוסתרים/מוצגים לפי טאב; רוחב מקסימלי 720px |

**שיפור אפשרי:** קישור `skip to main content`, והכרזה על טעינה (`aria-busy` על כפתור הניתוח בזמן `btn-loading`).

## 7. המלצות לפי עדיפות

### גבוהה

1. **למלא את `EN` ב־`templates.js`** בכל המפתחות הקיימים ב־`HE` (או fallback מפורש מ־HE) כדי למנוע שגיאות או פלט חלש בשפת פלט English.
2. **תיעוד הרצה:** לציין ב־README אם נדרש שרת סטטי ל־modules.

### בינונית

3. **בדיקות יחידה** ל־`normalize`, `evaluateRules`, `pickTemplate`, `decide` עם דוגמאות דוח קצרות.
4. **גרסה:** שדה `APP_VERSION` ב־`app.js` או הערה ב־footer לצורך תמיכה.

### נמוכה

5. **ייצוא JSON** של פלט ה־debug (כפתור “הורד debug.json”) לצוותים שרוצים לאוטומציה.
6. **הרחבת `extractTech`** (למשל Vue, Svelte, Python/FastAPI) לפי קהל היעד.
